# A股情绪指数 (eIndex) 项目开发规格文档

## 项目概述
eIndex是一个基于A股市场情绪指标的量化分析工具，通过计算全市场换手率、融资余额占比、涨停家数占比三个核心指标，生成综合情绪指数，为投资者提供客观的市场情绪参考。

## 核心策略

### Version
- Version number format：日期-XXX。例如：2026-03-10-099
- Version number显示在“eIndex A股情绪指数”右面
- 强制每次编译或修改网页（内容布局、样式、颜色，*.js），index.html和version number都需要更新。如果只是数据更新，index.html和version number则无需更新。



### 数据计算逻辑
每日收盘后（18:00后），待所有数据更新完毕，计算以下三个指标：

1. **全市场换手率** = 沪深两市总成交额 / 沪深两市总流通市值
2. **融资余额占比** = 融资余额 / 沪深两市总流通市值
3. **涨停家数占比** = 涨停股票数量 / 全市场股票总数

对每个指标计算其在过去250个交易日（约1年）中的**分位数**（0-100）：
- 分位数0：历史最低值
- 分位数100：历史最高值
- 当前分位数 = (当前值在历史数据中的排名位置) / 历史数据总数 × 100

**综合情绪指数** = (换手率分位数 + 融资余额占比分位数 + 涨停家数占比分位数) ÷ 3

### 交易信号规则
- **买入信号**：情绪指数 ≤ 20（市场极度恐慌）
- **卖出信号**：情绪指数 ≥ 80（市场极度狂热）
- **持有信号**：20 < 情绪指数 < 80（市场情绪正常）

### 数据源
- 腾讯财经API（实时性较好，免费）
- 东方财富网（备用）
- 聚宽/优矿（如需更精确数据，需API key）
- 由于API所属网站的限流和反爬措施，每次重新读取大量数据耗时很长或经常失败，所以当本地cache里有数据，就不需要重新用API读取数据。唯一的例外是当用户按“刷新”按钮时，需要重新用单日查询（非并发）逐日读取最近2个交易日的数据

### patch
- 做一个网络爬虫，从https://data.eastmoney.com/rzrq/sh.html读取缺失的193天（不包括今天)的深交所融资余额绝对值数据（单位：亿元），存在本地的sz_rz193.json
- 做一个网络爬虫，从https://www.sse.com.cn/market/view/和https://www.szse.cn/market/index.html读取上交所和深交所每天公布的“流通市值”（单位：亿元），存在本地的ltsz_cache.json
- 如果某天的上交所或深交所流通市值数据缺失，按照下列规则推算，从ltsz_raw.json重新生成ltsz_cache.json
  - 2016年：上交所占比 60.9%，深交所占比 39.1%
  - 2017年：上交所占比 63.8%，深交所占比 36.2%
  - 2018年：上交所占比 66.6%，深交所占比 33.4%
  - 2021年7月：上交所占比 59.8%，深交所占比 40.2%
  - 2021年8月：上交所占比 58.1%，深交所占比 41.9%
  - 2021年9月：上交所占比 58.8%，深交所占比 41.2%



### 指标卡片（实时显示）
- 当前情绪指数（大号数字，颜色渐变）
- 当前信号（买入/卖出/持有 + 图标）

### 今日三大指标值及分位数
- 交互图表（使用ECharts或Chart.js）
- 情绪指数历史走势图（可缩放时间轴）
- 数据从2015年开始
- 注意公共假日（A股不开盘的日期）不应该出现在统计中
- 买卖信号标记（在图表上标注信号点）
- 三大指标分位数对比图

### 数据表格
- 显示最近30个交易日数据
- 支持按日期、指数值、信号排序
- 提供CSV导出功能

### 采用动态分位数阈值，替代静态绝对值阈值
- 三大指标（成交额、融资余额、涨停家数）的分位数计算窗口，从250个交易日同步调整为120个交易日,不要hard code 120。
- 把买入、卖出改名为恐惧、贪婪。将固定的“贪婪/恐惧”阈值，改为基于滚动时间窗口（例如过去120个交易日）的动态分位数。
- 每次缩放时间范围的同时，要刷新Y轴的刻度，首先先保证贪婪值永远和红线左端水平，恐惧值永远和绿线左端水平，然后再调整其余Y轴刻度间隔，等比例缩放

### 刷新按钮
- 只要更新最近2天的数据，不花时间更新其余历史数据
- 不需要并发查询，采用单次查询，每次查询一天的数据，每个数据查询间隔1s
- 如果某天数据缺失，要在页面上提示缺失的具体数据类型和日期
- 更新最近2天的数据后，重新生成index.html页面上显示的所有曲线和表格
