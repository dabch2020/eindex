# eIndex 数据获取日志

## 数据缺失记录

### 1. 深交所融资余额
- **状态**: ❌ 无法获取
- **日期**: 2026-03-12 首次发现
- **现象**: 
  - `stock_margin_szse(start_date="20150101")` 报错：`got an unexpected keyword argument 'start_date'`
  - `stock_margin_detail_szse()` 报错：`Connection reset by peer`
- **影响**: 融资余额仅有上交所数据，缺少深交所部分
- **当前解决办法**: 使用上交所融资余额 × 1.67 估算全市场（沪深融资比例约 6:4）
- **后续改进**: 等 akshare 更新深交所接口后，调整调用参数重试

### 2. 融资余额近期数据延迟
- **状态**: ⚠️ 数据延迟
- **日期**: 2026-03-06 ~ 2026-03-12
- **现象**: 上交所融资数据更新有1-3个工作日延迟，最近几天 `margin_ratio` 为 0
- **影响**: 最近几天的 eIndex 仅由换手率和涨停两个指标计算（而非三个）
- **当前解决办法**: 脚本已处理——缺失指标时只用有数据的指标取均值
- **后续改进**: 无需额外处理，延迟数据会在后续更新中自动补入

### 3. 涨停数据历史覆盖有限
- **状态**: ⚠️ 部分缺失（持久缓存逐日积累中）
- **日期**: 2015-01-05 ~ 约2026-02-11 期间无涨停数据
- **现象**: `stock_zt_pool_em`（东方财富）仅保留最近约 1 个月的数据，更早的日期返回空
- **已验证的替代方案**:
  - Tushare Pro: token 权限不足（需充值积分），无法调用任何 API
  - AkShare 所有涨停相关接口: `stock_zt_pool_em`, `stock_zt_pool_previous_em`, `stock_zt_pool_strong_em`, `stock_market_activity_legu`, `stock_changes_em` 均无法获取历史涨停家数
- **影响**: 2015年至缓存起始日期期间的 eIndex 仅由换手率和融资余额两个指标计算
- **当前解决办法**: 
  1. 缺失指标时自动降级为可用指标的均值
  2. 采用**持久化缓存**策略 (`data/limitup_cache.json`)：每日运行自动保存当天涨停数据，随时间积累完整历史
- **缓存进度**: 2026-02-12 起开始积累（首次运行获取 15 天）
- **后续改进**: 无需额外操作，GitHub Actions 每日自动运行会持续扩展缓存覆盖范围
