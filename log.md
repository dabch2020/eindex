# eIndex 数据获取日志

## 算法（2026-03-17 更新）

### 概述

eIndex（情绪指数）通过 **4 个市场指标的分位数均值** 衡量A股市场情绪，范围 0–100。
低值 → 恐惧（买入信号），高值 → 贪婪（卖出信号）。

### 四大指标

| # | 指标 | 数据源 | 含义 |
|---|------|--------|------|
| 1 | **成交额分位** | 沪深指数日成交额（东方财富） | 市场活跃度 |
| 2 | **融资余额占比分位** | 沪深融资余额 / 流通市值 | 杠杆资金情绪 |
| 3 | **涨停家数占比分位** | 通达信 880006 涨停家数 / 全市场股票数 | 市场热度 |
| 4 | **市场方向分位** | 通达信 880008 全A等权指数 3 日收益率 | 市场涨跌方向 |

> 第 4 个指标（市场方向）于 2026-03-17 新增，解决原 3 指标无法区分「恐慌性抛售」和「贪婪性追涨」的问题——两者成交量/涨停数都高，但收益率方向相反。

### 计算流程

1. **分位数计算**：每个指标在最近 `PERCENTILE_WINDOW = 120` 个交易日内做滚动分位排名（0–100）
2. **eIndex 合成**：取 4 个分位数的**加权算术平均**
   ```
   eIndex = cje_pct × 0.20 + margin_pct × 0.05 + limitup_pct × 0.30 + return_pct × 0.35
   ```
3. **动态阈值**：在最近 120 个交易日的 eIndex 历史值中，取第 `FEAR_PERCENTILE = 15` 百分位作为恐惧线，第 `GREED_PERCENTILE = 84` 百分位作为贪婪线
4. **信号判定**：
   - `eIndex ≤ 恐惧线` → 🟢 恐惧（买入机会）
   - `eIndex ≥ 贪婪线` → 🔴 贪婪（卖出信号）
   - 其他 → 🟡 中性

### 关键参数

| 参数 | 值 | 说明 |
|------|----|------|
| `RETURN_LOOKBACK` | 3 | 880008 收益率回望天数 |
| `PERCENTILE_WINDOW` | 120 | 分位数滚动窗口（交易日） |
| `FEAR_PERCENTILE` | 15 | 恐惧线：eIndex 历史第 15 百分位 |
| `GREED_PERCENTILE` | 84 | 贪婪线：eIndex 历史第 84 百分位 |
| `W_CJE` | 0.20 | 成交额权重 |
| `W_MARGIN` | 0.05 | 融资余额权重（信号区分度最弱，降权） |
| `W_LIMITUP` | 0.30 | 涨停家数权重 |
| `W_RETURN` | 0.35 | 市场方向权重 |

### 参数优化

参数通过网格搜索 + 精细微调得出，基准：
- **人工标注**（权重 3x）：8 个恐惧日 + 8 个贪婪日（2022–2025）
- **880008 局部极值**（权重 1x）：自动检测的 50 个低点 + 46 个高点 = 96 个日期
- **容差**：±5 个交易日内命中即算正确
- **当前命中率**：恐惧 7/8，贪婪 7/8，总计 14/16（87.5%）

### 方法

使用 16 个人工标注的恐惧/贪婪日期，对比 eIndex 在该日是否发出对应信号。
- **精确命中**：当日 eIndex 信号与标注一致
- **±N日容差**：标注日前后 N 个交易日内出现对应信号即算命中

## 数据缺失记录

### 1. 融资余额数据源
- **状态**: ✅ 已解决
- **历史问题**:
  - `stock_margin_sse`：签名中 `end_date` 默认值硬编码为 `"20230922"`，导致 2023-09-25 之后 592 天融资数据缺失
  - `stock_margin_szse`：接口参数为 `date`（单日查询），不支持 `start_date`
  - `stock_margin_detail_szse`：报错 `Connection reset by peer`
- **解决方案**: 改用 `macro_china_market_margin_sh`（沪市）+ `macro_china_market_margin_sz`（深市）
  - 沪市数据 3867 条（2010-03-31 ~ 2026-03-11）
  - 深市数据 3669 条（2010-03-31 ~ 2026-03-11）
  - 覆盖率：2457 天中 2455 天有数据（99.9%）
- **深市补漏（2026-03-13）**:
  - `macro_china_market_margin_sz` 接口（东方财富转载深交所）存在大段缺失：
    - 2017-03 每周五（10/17/24）零星缺失 3 天
    - 2018-08-01 ~ 08-09 连续缺失 7 天
    - 2023-09-21 ~ 09-27 缺失 5 天
    - 2024-06-14 ~ 2025-06-18 大段断档约 178 天（深交所数据源问题）
  - 共 193 天深市缺失 + 1 天沪深均缺失(2017-03-13) = 194 天异常
  - 通过东方财富数据中心 API（`RPTA_WEB_RZRQ_LSSH`，`SCDM="001"`）批量获取深市融资余额真实数据，保存至 `data/sz_rz193.json`（193条），合并入 `data/margin_cache.json`
  - 2017-03-13（沪深均缺失）：通过同一API分别获取沪市（`SCDM="007"`）和深市数据补齐
  - 补漏后数据中半值异常从 194 天降为 **0 天**，所有日期均为真实数据
- **剩余缺失**:
  - 最新交易日：融资数据通常延迟1个工作日发布，脚本已处理（缺失指标时只用有数据的指标取均值）

### 2. 涨停数据历史覆盖

- **解决方案**: 改用 mootdx 库通过通达信行情协议获取 **880006 停板家数** 指标
  - `close` 列 = 涨停家数，`open` 列 = 跌停家数
  - 一次性获取 **2457 天** 历史数据（2016-01-26 至今）
  - 每日增量缓存到 `data/limitup_cache.json`
- **起始日对齐**: eIndex 起始日已调整为 2016-01-26，与 880006 最早数据对齐，三大指标从第一天起全部可用

### 方案 A：调整因子权重（推荐，改动最小效果最大）

当前四个因子等权平均（各25%），但 margin 对情绪极端的捕捉能力最弱。建议降低 margin 权重：

```
eIndex = cje_pct × 0.30 + limitup_pct × 0.30 + ret_pct × 0.25 + margin_pct × 0.15
```

预期效果：恐惧日 eIndex 大幅下降（margin 高值的拖累减少），贪婪日 eIndex 上升（margin 低值的拖累减少），±5天命中率从 56% 提升到约 70%。

### 方案 B：增加新指标 — 跌停家数

目前只用涨停家数（880006 close 列），但恐惧日的核心特征是**跌停家数激增**。880006 的 `open` 列就是跌停家数，已有数据源，只需额外缓存：

```
跌停比例 = 跌停家数 / 全市场股票数
```

恐惧信号 = 跌停分位数高 + 成交额分位数低 → 精确捕捉恐慌抛售。

### 方案 C：融资余额指标改为"变化率"

用 `margin_ratio 的 N 日变化率` 替代 `margin_ratio 绝对值占比`。市场恐慌时融资余额快速下降（强平/止损），用变化率能更灵敏地反映情绪转折：

```
margin_change = (今日margin_ratio - N日前margin_ratio) / N日前margin_ratio
```

恐慌时 margin_change 为负（融资快速撤出），狂热时 margin_change 为正（融资快速涌入），比绝对值占比更能反映情绪变化方向。
